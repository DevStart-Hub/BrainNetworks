In the last tutorial we found the combination of η and γ that best describes the average brain across all 20 participants. But that’s just the beginning of the story!

Think about it: we’ve been summarising 20 different brains with a single pair of numbers. In reality, each brain follows its own wiring rules. Some people might have a stronger distance penalty, meaning their brains are built more locally. Others might show stronger homophily, meaning their brains have tighter clusters. And if those individual differences in wiring rules are meaningful, they might relate to something biologically or psychologically important — like early life stress.

That’s the big question for this tutorial: do people who experienced more early life stress grow their brain networks using different wiring rules?

The plan

The approach is straightforward:

  1. Take the group optimum we found in the last tutorial as our starting point
  2. Run a new, finer sweep centred on that optimum
  3. For each brain, extract the (η, γ) that best fits that individual
  4. Correlate those individual estimates with early life stress scores

The key difference from the last tutorial is step 3: instead of averaging energy across all brains and picking one winner, we let each brain pick its own winner. Let’s walk through it!

Load the data

We start exactly as before: load the 20 preprocessed brains and the distance matrix. This time we also load the early life stress scores, one per participant.

import numpy as np
import torch

brains = np.load("brain_networks_20_preprocessed.npy")
num_brains, num_nodes, _ = brains.shape
binary_brains = (brains > 0).astype(float)
binary_brains_tensor = torch.tensor(binary_brains, dtype=torch.float32)

dist_np = np.load("distance_matrix.npy")
distance_matrix = torch.tensor(dist_np, dtype=torch.float32)

stress_scores = np.load("stress_scores.npy")

num_connections = int(binary_brains_tensor[0].sum().item() / 2)
print(f"Loaded {num_brains} brains and {num_brains} stress scores")

Find the group optimum

We then load the group-level experiments we saved in the last tutorial, and extract the group optimum. We need it to centre our new, finer sweep!

import pickle
import numpy as np
from gnm import evaluation

with open("experiments.pkl", "rb") as f:
    group_experiments = pickle.load(f)

criteria = [
    evaluation.DegreeKS(),
    evaluation.BetweennessKS(),
    evaluation.ClusteringKS(),
    evaluation.EdgeLengthKS(distance_matrix),
]
energy = evaluation.MaxCriteria(criteria)
energy_key = str(energy)

mean_energies = [
    float(exp.evaluation_results.binary_evaluations[energy_key].mean())
    for exp in group_experiments
]

best_idx    = int(np.argmin(mean_energies))
best_exp    = group_experiments[best_idx]
group_eta   = float(best_exp.run_config.binary_parameters.eta)
group_gamma = float(best_exp.run_config.binary_parameters.gamma)

print(f"Group optimum: eta = {group_eta:.2f}, gamma = {group_gamma:.2f}")

The fine-grained sweep

Now for the new sweep. The group sweep covered a wide region of the parameter space (η from -5 to 0, γ from -0.5 to 0.5). For individual fitting, we do something smarter: we zoom in on the region around the group optimum.

Why is this better? Two reasons:

  • Precision: a finer grid means we can pick up subtle differences between individuals that would be invisible on a coarse grid.
  • Speed: by covering a smaller range, we can afford more grid points and more simulations, giving us more reliable estimates.

We also run more simulations than before — remember, GNMs are probabilistic, so more simulations means more stable estimates for each individual:

from gnm import fitting, generative_rules

n_eta   = 15
n_gamma = 15
n_sims  = 20

eta_values   = torch.linspace(group_eta   - 2.0, group_eta   + 2.0, n_eta).clamp(-7, 0)
gamma_values = torch.linspace(group_gamma - 0.3, group_gamma + 0.3, n_gamma).clamp(-1, 2)

binary_sweep_parameters = fitting.BinarySweepParameters(
    eta    = eta_values,
    gamma  = gamma_values,
    distance_relationship_type         = ["powerlaw"],
    preferential_relationship_type     = ["powerlaw"],
    generative_rule  = [generative_rules.MatchingIndex()],
    num_iterations   = [num_connections],
)

sweep_config = fitting.SweepConfig(
    binary_sweep_parameters = binary_sweep_parameters,
    num_simulations         = n_sims,
    distance_matrix         = [distance_matrix],
)

We then run the sweep in exactly the same way as before:

import time

start = time.perf_counter()

experiments = fitting.perform_sweep(
    sweep_config         = sweep_config,
    binary_evaluations   = [energy],
    real_binary_matrices = binary_brains_tensor,
    save_model           = False,
    save_run_history     = False,
)

elapsed = time.perf_counter() - start
print(f"Sweep complete in {elapsed:.1f} s")

Extracting individual estimates

Here is where the magic happens. Instead of taking the mean energy across all brains and finding one global winner, we now ask: for each brain separately, which (η, γ) gives the lowest energy?

The GNM toolbox has a function for exactly this: optimise_evaluation(). You’ve seen it before! But now, instead of thinking about it as a way to find the best overall configuration, notice what it actually returns: one experiment per brain — specifically, the parameter configuration that best fits that individual.

optimal_experiments, optimal_energies = fitting.optimise_evaluation(
    experiments = experiments,
    criterion   = energy,
)

Now we just read off the (η, γ) from each brain’s best-fitting experiment:

import pandas as pd

rows = []
for brain_idx, (opt_exp, opt_energy) in enumerate(zip(optimal_experiments, optimal_energies)):
    rows.append({
        "brain":       brain_idx,
        "eta":         float(opt_exp.run_config.binary_parameters.eta),
        "gamma":       float(opt_exp.run_config.binary_parameters.gamma),
        "best_energy": float(opt_energy),
        "stress":      float(stress_scores[brain_idx]),
    })

df_ind = pd.DataFrame(rows)
print(df_ind)

We now have a dataframe with one row per participant, telling us their individual η, γ, best energy, and stress score.

Correlating with early life stress

Now for the key question: do individual differences in wiring rules relate to early life stress?

We test this with a simple Pearson correlation — the same kind we used in the Topology tutorial. We check both parameters separately:

from scipy import stats

r_eta,   p_eta   = stats.pearsonr(df_ind["stress"], df_ind["eta"])
r_gamma, p_gamma = stats.pearsonr(df_ind["stress"], df_ind["gamma"])

print(f"eta   vs stress: r = {r_eta:.2f},  p = {p_eta:.3f}")
print(f"gamma vs stress: r = {r_gamma:.2f}, p = {p_gamma:.3f}")

And we visualise the results with scatter plots:

import matplotlib.pyplot as plt

BLUE = "#2E86AB"

fig, axes = plt.subplots(1, 2, figsize=(10, 4))

for ax, param, r_val, p_val, ylabel in zip(
    axes,
    ["eta",               "gamma"],
    [r_eta,               r_gamma],
    [p_eta,               p_gamma],
    ["η (distance penalty)", "γ (homophily)"],
):
    ax.scatter(df_ind["stress"], df_ind[param], color=BLUE, s=70, zorder=3)
    m, b = np.polyfit(df_ind["stress"], df_ind[param], 1)
    xs = np.linspace(df_ind["stress"].min(), df_ind["stress"].max(), 100)
    ax.plot(xs, m * xs + b, color="red", lw=1.8)
    ax.set_xlabel("Early life stress score", fontsize=12)
    ax.set_ylabel(ylabel, fontsize=12)
    ax.set_title(f"r = {r_val:.2f},  p = {p_val:.3f}", fontsize=13)
    ax.spines[["top", "right"]].set_visible(False)

plt.tight_layout()
plt.savefig("../images/GenerativeModels/individual_correlations.png", dpi=150)
plt.show()

What do the results mean?

The two panels tell us whether early life stress is linked to differences in wiring rules. Let’s think about what each parameter means biologically:

A correlation with η would mean that people with more early life stress have a different spatial wiring strategy. A more negative η means the brain is more strongly constrained by distance — it mostly connects nearby regions. A less negative η means the brain is more willing to form long-range connections. So if stress correlates with η, it would suggest that early adversity changes how much the developing brain “cares about” physical distance when laying down its connections.

A correlation with γ would mean that stress is linked to differences in homophily — how strongly the brain’s wiring builds on itself, connecting regions that already share neighbours. Stronger homophily builds tighter, more segregated modules; weaker homophily produces a more integrated, less clustered network. A stress-related shift in γ would suggest that adversity affects how modular the brain becomes during development.

ImportantWhat if the correlations are not significant?

That is a completely valid outcome! Remember: we’ve been working with only 20 brains, which gives us limited statistical power. Real studies in this area typically use hundreds of participants. With 20 people, you would need a very strong correlation to reach significance.

What matters here is understanding the logic: we’ve moved from describing the brain with a handful of topology metrics to describing it with two interpretable wiring parameters. And whether the result is significant or not, we’re asking a much richer question than we could with topology alone.

NoteFrom two numbers to a whole story

It’s worth stepping back and appreciating what we’ve done across these tutorials. We started with raw connectivity matrices and ended up with two numbers per person that capture how their brain wires itself. Those numbers are not arbitrary: they describe real developmental processes, they have a biological interpretation, and they can be compared across individuals and related to life experiences.

That’s the power of Generative Network Models!

So our research question is: **do people who experienced more early life stress end up having brains wired differently?**

Now we can finally tackle it head on!

## Our 20 brains

Let's start by reminding ourselves of what we are working with. Here are the 20 brain networks we have been working with throughout these tutorials:

Each panel shows a different individual's connectivity matrix. They all look similar at a glance, but the differences between them are exactly what we want to capture. The key question is: are those differences related to early life stress?

## From group to individual

In the last tutorial, we averaged energy across all 20 brains to find the single best-fitting (η, γ). Now we want to be more specific: for *each brain separately*, we pick the parameter configuration that gave that brain the lowest energy.

We don't need to run a new sweep for this! All the info is already in the experiments from last time. Let's load them:

``` python
import pickle
import numpy as np
import torch
import pandas as pd
import matplotlib.pyplot as plt
from collections import Counter

with open("experiments.pkl", "rb") as f:
    group_experiments = pickle.load(f)
```

For each brain, we find the config with the lowest energy by averaging the energy tensor over simulations and then taking the argmin across configs. We also rebuild the landscape and group optimum — only needed so we can overlay them on the plot:

``` python
# Per-brain coarse optima
per_brain_energy  = torch.stack(
    [exp['energy_tensor'].mean(dim=0) for exp in group_experiments]
)  # [n_configs, n_brains]
best_config_per_brain = per_brain_energy.argmin(dim=0)

ind_eta_c   = [group_experiments[int(idx)]['eta']   for idx in best_config_per_brain]
ind_gamma_c = [group_experiments[int(idx)]['gamma'] for idx in best_config_per_brain]

# Rebuild landscape and group optimum for plotting
mean_energies_group = [float(exp['energy_tensor'].mean()) for exp in group_experiments]
rows_group = [{"eta": exp['eta'], "gamma": exp['gamma'], "energy": e}
              for exp, e in zip(group_experiments, mean_energies_group)]
landscape   = pd.DataFrame(rows_group).pivot(index="eta", columns="gamma", values="energy")
best_idx    = int(np.argmin(mean_energies_group))
group_eta   = group_experiments[best_idx]['eta']
group_gamma = group_experiments[best_idx]['gamma']

counts = Counter(zip(ind_eta_c, ind_gamma_c))

fig, ax = plt.subplots(figsize=(7, 5))
im = ax.imshow(
    landscape.values,
    origin="lower", aspect="auto", cmap="viridis_r",
    extent=[
        landscape.columns.min(), landscape.columns.max(),
        landscape.index.min(),   landscape.index.max(),
    ],
)
for (eta_val, gamma_val), n in counts.items():
    ax.scatter(gamma_val, eta_val, marker="x", s=140, color="white",
               linewidths=2.5, zorder=5)
    if n > 1:
        ax.text(gamma_val + 0.02, eta_val + 0.1, str(n),
                color="white", fontsize=9, fontweight="bold", zorder=6)
ax.scatter(group_gamma, group_eta, s=150, color="red", zorder=7, label="Group optimum")
plt.colorbar(im, ax=ax, label="Energy (max KS)")
ax.set_xlabel("gamma (homophily)", fontsize=12)
ax.set_ylabel("eta (distance penalty)", fontsize=12)
ax.set_title("Individual optima on coarse grid", fontsize=13)
ax.legend(fontsize=10)
plt.tight_layout()
plt.savefig("../images/GenerativeModels/individual_landscape_coarse.png", dpi=150)
plt.show()
```

![](../images/GenerativeModels/individual_landscape_coarse.png){width="80%" fig-align="center"}

Do you see the problem? The white numbers next to several markers show how many participants are stacked at the exact same point! With a coarse 10×10 grid, many individuals end up sharing the same (η, γ) even though their true optima may be slightly different. **We are losing precision.**

## A finer sweep

To fix this, we run a second, more targeted sweep. Looking at the coarse landscape, the interesting action is clearly happening in a narrower region: roughly η between −3 and −0.5, and γ between 0 and 0.5. So that's exactly where we sweep:

::: callout-important
## Choose the sweep range carefully!

There is no automatic rule for picking the fine-sweep range. We chose η ∈ \[−3, −0.5\] and γ ∈ \[0, 0.5\] by eye, based on where the coarse landscape placed most individual optima. For your own dataset, you will need to do the same inspection.

One thing that is **critical**: after extracting the per-brain optima, make sure no participant ends up with their best point right on the *border* of the parameter space. If someone's best (η, γ) is at the minimum or maximum value you tested, their true optimum might be *beyond* that boundary and you are artificially clamping them. When this happens, widen the range in that direction and re-run.
:::

For this we need to load the brain networks, distance matrix, and stress scores:

``` python

```

We increased `n_eta` and `n_gamma` to 20 each (instead of 10), and we also increased `n_sims` to 20. More simulations mean more stable estimates for each individual, which is important when we're trying to detect subtle differences between people.

    {

## What do the results mean?

The two panels tell us whether early life stress is linked to individual differences in wiring rules:

**A correlation with η** would mean that people with more early life stress have a different *spatial* wiring strategy. A less negative η means the brain is more willing to form long-range connections, which is consistent with the Adaptive Stochasticity Hypothesis: adversity might loosen the distance constraint, allowing for a more diffuse, stochastic developmental process.

**A correlation with γ** would mean that stress is linked to differences in *homophily*. Weaker homophily produces a more integrated, less clustered network — again consistent with a more stochastic developmental programme that does not lock in tight local communities.
